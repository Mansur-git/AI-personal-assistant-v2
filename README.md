# Yaara AI

**Yaara AI** is a system-oriented personal AI assistant designed to explore how **production-style agentic AI systems** can be built using structured architecture, controlled execution, and tool-driven workflows.

Unlike typical prompt-based assistants, Yaara treats the language model as **one component in a larger system**, focusing on reliability, modular design, and clear separation between reasoning and execution.

This project serves as an experimentation platform for building **scalable and production-minded GenAI systems**.

---

# Overview

Most AI assistants rely heavily on prompt chains and monolithic pipelines. Yaara AI takes a **systems-first approach**, emphasizing explicit control flow, modular orchestration, and observability.

The goal is to explore how AI assistants can evolve from **demo-level tools to reliable production systems**.

Yaara focuses on:

* structured reasoning workflows
* tool-based execution
* system observability
* architecture-level trade-offs

---

# Key Features

* Agent-based workflow orchestration

# System Architecture

Yaara AI follows a **system-oriented architecture** where the language model operates within a controlled environment.

The system separates:

* reasoning
* tool execution
* memory management
* orchestration

This design allows experimentation with **reliable and debuggable agentic AI systems**.

# Tech Stack

### AI / Agent Frameworks

* **LangGraph** – agent workflow orchestration
* **LangChain** – LLM integration and tool interfaces

### Observability

* **Langfuse** – tracing, logging, and monitoring AI system behavior

### Memory & Context

* **Mem0** – memory layer for persistent contextual interactions

### Models

* **Phi-3 Mini**
* **OpenAI API**

### Backend

* **FastAPI** – backend API and system orchestration

### Databases

* **MongoDB** – application data storage
* **Neo4j** – knowledge graph relationships
* **QuadrantDB** – vector storage for semantic retrieval
